## Project

Didymus is a Hugo-based static site (Biblical Science Accessibility) built on the [Hinode](https://gethinode.com) theme. Deployed to Netlify. Default content language is Dutch (`nl`); English (`en`) is the secondary locale.

## Commands

Dev server (vendors Hugo modules via `prestart`, then serves on :1313):

```bash
npm start              # development
npm run start:prod     # production env
```

Build:

```bash
npm run build          # hugo --gc --minify (runs clean:public + mod:vendor first)
npm run build:debug    # hugo -e debug --debug
npm run build:headers  # regenerate netlify.toml + config/_default/server.toml from Hugo (see note)
```

Lint (all three run by `npm test`):

```bash
npm run lint:scripts   # eslint assets/js
npm run lint:styles    # stylelint assets/scss
npm run lint:markdown  # markdownlint-cli2 on *.md + content/**/*.md
```

Hugo modules:

```bash
npm run mod:vendor     # rewrite _vendor from go.mod (required before build/serve)
npm run mod:update     # upgrade all Hugo modules, re-vendor, tidy
npm run mod:tidy
```

Version check: `npm run check` (Hugo) / `npm run env`.

## Architecture

### Hugo + Hinode + Go modules

Site content and config live in this repo; theme/behavior comes from Hugo modules listed in `go.mod`. Key imports: `github.com/gethinode/hinode/v2` (theme) and `github.com/gethinode/mod-blocks` (content blocks). All modules are vendored into `_vendor/` — **do not edit `_vendor/` directly**; change `go.mod` and run `npm run mod:vendor`.

Config hierarchy (`config/_default/`):

- `hugo.toml` — core site config, output formats, module mounts/imports, CSP segment definition
- `languages.toml`, `params.toml`, `params.nl.toml` — i18n + theme params
- `server.toml` — dev server headers (auto-generated by `build:headers:dev`)
- `config/ci/` — CI-specific overrides

### Content

`content/nl/` and `content/en/` mirror the site tree. Sections include `about`, `contact`, `content`, `languages`, `software`, `testimonials`. Hinode drives layouts; only local overrides live in `layouts/`.

### Netlify deploy flow

`netlify.toml` is **auto-generated** — the comment at the top warns not to edit it. It is produced by Hugo rendering the `headers` segment (see `[segments.headers]` in `hugo.toml`, output formats `netlify` + `REDIR`) via `npm run build:headers:prod`, which copies `prebuild-headers-prod/netlify.toml` to the repo root. Same flow for `server.toml` (dev). To change headers/CSP/redirects, edit the Hugo templates that render those outputs, then regenerate.

Netlify build runs `npm run build` (prod) and applies two plugins: `@gethinode/netlify-plugin-dartsass` and `netlify-plugin-hugo-cache-resources`.

### Assets

Styling/JS come from Hinode modules. Add project overrides in `assets/js/` and `assets/scss/`; they compile through Hugo Pipes + PostCSS (`config/postcss.config.js`) and Dart Sass.

### CI

`.github/workflows/lint.yml` — runs `npm run lint` on push/PR.
`.github/workflows/mod-update.yml` — scheduled Hugo module updates.
`.github/workflows/auto-merge.yml` — auto-merge bot PRs.

Pre-commit hooks via `.pre-commit-config.yaml` enforce Conventional Commits).
